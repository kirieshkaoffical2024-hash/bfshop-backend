## 🏠 Локальная сеть (LAN)

### Если друзья в одной сети Wi-Fi с вами:

#### Шаг 1: Узнайте свой IP адрес

**Windows:**
```bash
ipconfig
```
Найдите строку "IPv4 Address" - это ваш локальный IP (например: `192.168.1.100`)

#### Шаг 2: Обновите конфигурацию

1. Откройте `BFshop/backend/.env`
2. Измените:
```
DATABASE_URL=postgresql://postgres:2306@localhost:5432/bfshop
PORT=3000
```

3. Откройте `BFshop/frontend/assets/js/app.js`
4. Измените:
```javascript
const API_URL = 'http://ВАШ_IP:3000/api';
// Например: const API_URL = 'http://192.168.1.100:3000/api';
```

#### Шаг 3: Откройте порты в брандмауэре

**Windows Firewall:**
1. Откройте "Панель управления" → "Брандмауэр Windows"
2. Нажмите "Дополнительные параметры"
3. Выберите "Правила для входящих подключений"
4. Нажмите "Создать правило"
5. Выберите "Порт" → Далее
6. TCP, порты: `3000, 8080` → Далее
7. Разрешить подключение → Далее
8. Все профили → Далее
9. Имя: "BFshop" → Готово

#### Шаг 4: Запустите сервер

```bash
БЫСТРЫЙ_ЗАПУСК.bat
```

#### Шаг 5: Друзья подключаются

Друзья открывают браузер и вводят:
```
http://ВАШ_IP:8080
```
Например: `http://192.168.1.100:8080`

## 🌍 Интернет (через ngrok)

### Для доступа из любой точки мира (бесплатно):

#### Шаг 1: Установите ngrok

1. Скачайте: https://ngrok.com/download
2. Распакуйте в папку `C:\ngrok`
3. Зарегистрируйтесь на ngrok.com
4. Получите authtoken

#### Шаг 2: Настройте ngrok

```bash
cd C:\ngrok
ngrok config add-authtoken ВАШ_ТОКЕН
```

#### Шаг 3: Запустите BFshop

```bash
БЫСТРЫЙ_ЗАПУСК.bat
```

#### Шаг 4: Запустите ngrok для backend

Откройте новое окно командной строки:
```bash
cd C:\ngrok
ngrok http 3000
```

Вы получите URL типа: `https://abc123.ngrok.io`

#### Шаг 5: Запустите ngrok для frontend

Откройте еще одно окно:
```bash
cd C:\ngrok
ngrok http 8080
```

Вы получите URL типа: `https://xyz789.ngrok.io`

#### Шаг 6: Обновите конфигурацию

1. Откройте `BFshop/frontend/assets/js/app.js`
2. Измените:
```javascript
const API_URL = 'https://abc123.ngrok.io/api';
```

#### Шаг 7: Друзья подключаются

Друзья открывают:
```
https://xyz789.ngrok.io
```

⚠️ **Важно:** При каждом перезапуске ngrok URL меняется!

## ☁️ Облачный хостинг (постоянный адрес)

### Railway.app (рекомендуется)

#### Шаг 1: Создайте аккаунт
- Зайдите на https://railway.app
- Войдите через GitHub

#### Шаг 2: Создайте проект
1. New Project → Deploy PostgreSQL
2. Скопируйте DATABASE_URL

#### Шаг 3: Загрузите backend
1. New → GitHub Repo
2. Выберите папку `BFshop/backend`
3. Добавьте переменные окружения:
   - `DATABASE_URL` = (из шага 2)
   - `JWT_SECRET` = `your-secret-key-here`
   - `PORT` = `3000`

#### Шаг 4: Загрузите frontend
1. New → Static Site
2. Выберите папку `BFshop/frontend`
3. Обновите `app.js`:
```javascript
const API_URL = 'https://ваш-backend.railway.app/api';
```

#### Шаг 5: Готово!
Друзья используют URL вашего frontend:
```
https://ваш-frontend.railway.app
```

### Heroku (альтернатива)

#### Шаг 1: Установите Heroku CLI
```bash
npm install -g heroku
```

#### Шаг 2: Войдите
```bash
heroku login
```

#### Шаг 3: Создайте приложение
```bash
cd BFshop/backend
heroku create bfshop-backend
heroku addons:create heroku-postgresql:hobby-dev
```

#### Шаг 4: Деплой backend
```bash
git init
git add .
git commit -m "Initial commit"
git push heroku master
```

#### Шаг 5: Деплой frontend
```bash
cd ../frontend
heroku create bfshop-frontend
# Обновите API_URL в app.js
git init
git add .
git commit -m "Initial commit"
git push heroku master
```

## 🔒 Безопасность

### Рекомендации:

1. **Измените JWT_SECRET** в `.env`:
```
JWT_SECRET=ваш-очень-длинный-случайный-ключ-12345
```

2. **Измените пароль PostgreSQL**:
```sql
ALTER USER postgres WITH PASSWORD 'новый_сложный_пароль';
```

3. **Используйте HTTPS** (ngrok и облачные сервисы делают это автоматически)

4. **Ограничьте доступ** к базе данных (только с вашего сервера)

## 🆘 Проблемы и решения

### "Failed to fetch"
- Проверьте, что backend запущен
- Проверьте API_URL в app.js
- Проверьте брандмауэр

### "Connection refused"
- Проверьте, что PostgreSQL запущен
- Проверьте DATABASE_URL в .env

### "CORS error"
- Добавьте в `server.js`:
```javascript
app.use(cors({
    origin: '*' // или конкретный домен
}));
```
